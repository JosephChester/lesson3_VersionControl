Lesson 3 Highlights

Joseph Chester

Highlight #1 Version Control Systems

Version Control Systems are leveraged to manage the various states of software design. This is a crucial piece of software development as it allows for applications to be updated and changed while allowing for the retention of previous version in the event that the changes break the application, cause undesirable results or add new bugs. Version Control is also referred to as source control and function by placing timestamps and numbers on each package once the development is ready for testing. This process came about to solve the need for a logical system that manages changes within application or system code.

Highlight #2 Local Version Control Systems

Local Version Control Systems (LVCS) are a type of source control that is housed on one machine and typically only utilized by one developer. This method of version control is created to house a local database that records code changes and versions on the developer’s machine. This method is prone to user error because they have to manually specify where the files need to be stored and can on occasion overwrite existing data. Another problem with this method is the risk of data loss. While it is easy to setup and manage, the code versions are only stored on one machine and if there is a hard drive failure and if the data is not being backed up you risk losing all of your work.

Highlight #3 Centralized Version Control Systems

Centralized Version Control System (CVCS) was created to allow multiple developers access to the same code source. This provided the opportunity for multiple points of development to occur simultaneously and greatly improved efficiency regarding development speed. Another benefit with this system was the allowance for administrators to manage one single database instead of keeping track of developer’s individual databases. This is useful for restricting access and monitoring usage. Like LVCS though, having all of the code stored in one location is a huge risk and in the event that the database is corrupted and no usable backups are available, you will lose all versioning associated with code changes.

Highlight #4 Distributed Version Control Systems

Distributed Version Control Systems (DVCS) currently provides the most flexibility and redundancy for maintaining code and previous code changes. When developers check out code using a DVCS not only do they check out the current version, but previous versions are also included. This feature allows for redundancy in the event of a database failure because all checked out versions include past history removing the single point of failure. DVCS also allows for multiple users to check out code and simultaneously work on the same project, assisting in their ability meet deadlines and produce completed applications more quickly. Git is an example of DVCS.

Highlight #5 Git

Git is a program that differs from others in its ability to compile multiple snapshots of the applications versioning into one file. If no changes were checked in or made it will not update the version number on the file being viewed. One of the most powerful features of Git is its ability to work offline, this will allow for users to update code when they are unable to access the internet and then check in and commit their changes when they are able. Like any other DVCS multiple users can check out the application to work on simultaneously. Git functions in three basic states, the working directory where a user has checked out the files to be modified, a staging area where changes have been submitted but not committed to the database and the .git directory or repository where changes are committed.
